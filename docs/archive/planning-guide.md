# CLIPBee 작업 지침

이 폴더는 `CLIPBee` 제작을 위한 기획/디자인 워크스페이스다. 작업자는 기능 구현, 디자인 구현, 이미지 생성, 문서 보강을 시작하기 전에 이 파일을 먼저 읽고 따른다.

## 1. 반드시 읽을 문서

작업 시작 전 아래 순서로 확인한다.

1. `PRD.MD`
2. `design.md`
3. `output/playwright/` 캡처 파일

`PRD.MD`는 기능 기준이고, `design.md`는 화면/컴포넌트 기준이다. 두 문서가 충돌하면 `PRD.MD`의 기능 요구사항을 우선하고, 시각 표현은 `design.md`를 따른다.

## 2. 응답 규칙

- 코딩 관련 답변에서 코드를 길게 출력하지 않는다.
- 결론부터 짧게 말한다.
- 선택지가 있으면 `하기 전 상태`와 `고치면 바뀌는 점`을 구분한다.
- 사용자의 말이 틀렸으면 맞장구치지 말고 짧게 지적한다.
- 명령 실행 결과는 핵심만 요약한다.
- 구현이 끝나면 변경 파일과 검증 결과만 간결하게 보고한다.

## 3. 제품 방향

CLIPBee는 편집몬의 기능 구조를 참고하되, 촬영 분야 전용 플랫폼으로 재설계한다.

핵심 서비스:

- 촬영자 모집
- 촬영자 프로필
- 커뮤니티
- 공지사항
- 스토어
- 스토어 등록
- 기업 인증
- 유료 노출 상품
- 연락처 열람권
- 자동점프
- 추천 프로필

## 4. 절대 금지

- 편집몬 로고, 이미지, 배너, 아이콘, 문구를 그대로 복사하지 않는다.
- 편집몬 게시글 원문을 샘플 데이터로 그대로 쓰지 않는다.
- 기존 사업자 정보, 계좌번호, 대표자명, 주소를 CLIPBee에 넣지 않는다.
- 원본 사이트 캡처를 실제 서비스 이미지로 쓰지 않는다.
- 이미지 안에 깨지는 한글 텍스트를 생성하지 않는다.
- 기능을 줄여서 “비슷하게” 만들지 않는다. 기능 구조는 PRD 기준으로 맞춘다.

UI 규칙:

- 카드 안에 큰 카드를 중첩하지 않는다.

- 모바일은 카드 가독성을 우선한다.
- 버튼 텍스트는 짧게 쓴다.
- 프리미엄/추천/공지/마감 상태는 배지로 표시한다.

## 8. 이미지 프리셋 전체 규칙

CLIPBee는 이미지가 중요한 서비스다. 구현 시 실제 이미지 파일이 없으면 아래 프리셋 기준으로 생성 이미지 또는 플레이스홀더를 만든다.

공통 규칙:

- 원본 편집몬 이미지를 사용하지 않는다.
- 이미지 안에 글자나 로고를 넣지 않는다.
- 촬영 현장, 장비, 조명, 카메라, 인물 실루엣, 스튜디오, 결과물 느낌을 중심으로 만든다.
- 사람 얼굴은 특정 실존 인물처럼 보이지 않게 한다.
- 너무 어둡거나 흐린 이미지를 피한다.
- 카드 썸네일은 모바일에서 잘려도 핵심 피사체가 중앙에 남아야 한다.
- 모든 이미지는 WebP 우선, 필요 시 PNG 대체.
- alt 텍스트를 반드시 둔다.

권장 저장 위치:

- `public/images/presets/jobs/`

- `public/images/presets/store/`

- `public/images/presets/placeholders/`
- `public/images/presets/banners/`

파일명 규칙:

- `shootmon-hero-desktop-01.webp`

- `shootmon-job-brand-01.webp`

## 9. 필수 이미지 프리셋

### 9.1 메인 히어로/상단 배너

desktop:

- 크기: 1440x360
- 용도: PC 메인 상단 배너
- 방향: 촬영 현장 전체 분위기, 카메라/조명/스태프 실루엣, 밝은 상업 촬영장

mobile:

- 크기: 780x360
- 용도: 모바일 상단 배너
- 방향: 중앙 피사체가 잘리지 않는 촬영 현장

프롬프트 방향:

- commercial video production set
- camera rig and softbox lighting
- professional Korean production studio
- warm orange accent
- clean realistic photography
- no text, no logo

### 9.2 촬영자 모집 카드

기본 크기:

- 640x360

필수 카테고리:

- 브랜드/광고 촬영
- 유튜브/채널 촬영
- 숏폼/Reels/TikTok
- 웨딩 촬영
- 행사/컨퍼런스 촬영
- 제품/커머스 촬영
- 인터뷰/다큐 촬영
- 라이브/스트리밍
- 드론 촬영
- 스튜디오 촬영
- 부동산/공간 촬영
- 뷰티/패션 촬영
- 교육/강의 촬영
- 스포츠/공연 촬영

각 카테고리마다 최소 2장씩 준비한다.

기본 크기:

- 800x600

스타일:

- 촬영 장비와 포트폴리오 결과물이 보이는 이미지
- 인물은 뒷모습/측면/손/실루엣 위주
- 얼굴 중심 초상화보다 작업 전문성이 드러나야 한다.

필수 프리셋:

- 광고 촬영감독
- 웨딩 촬영자
- 드론 촬영자
- 제품 촬영자
- 라이브 송출팀
- 인터뷰 촬영자
- 패션 촬영자
- 행사 촬영팀
- 스튜디오 운영자

### 9.4 프로필 아바타

기본 크기:

- 256x256

규칙:

- 실존 인물과 닮은 얼굴 생성 금지
- 기본 아바타는 카메라 렌즈, 조리개, 실루엣 기반으로 만든다.
- 남/여/팀/기업 기본 아바타를 구분한다.

필수:

- 개인 촬영자 기본
- 여성 촬영자 기본
- 남성 촬영자 기본
- 촬영팀 기본
- 기업회원 기본

### 9.5 스토어 상품 썸네일

기본 크기:

- 640x480

필수 카테고리:

- 촬영 서비스 패키지
- 드론 촬영
- 스튜디오 대관
- 장비 대여
- 조명/오디오 패키지
- 라이브 송출
- LUT/프리셋
- 계약서/가이드 템플릿
- 제품 촬영
- 웨딩 촬영

규칙:

- 가격이나 홍보 문구를 이미지 안에 넣지 않는다.
- 상품명/가격은 UI 텍스트로 표시한다.

### 9.6 커뮤니티/공지 썸네일

기본 크기:

- 640x360

필수:

- 자유게시판 기본
- 촬영 피드백 기본
- 공모전 기본
- 촬영랩 기본
- 공지사항 기본
- 이벤트 기본

### 9.7 플레이스홀더

필수:

- 이미지 없음
- 프로필 없음
- 회사 로고 없음
- 스토어 이미지 없음
- 배너 준비중
- 심사중
- 비공개

규칙:

- 회색 박스만 쓰지 않는다.
- 카메라/프레임/조리개 아이콘을 약하게 넣는다.
- 텍스트는 UI에서 표시하고, 이미지 자체에는 넣지 않는다.

## 11. 이미지 생성 품질 기준

통과 기준:

- 피사체가 선명하다.
- 촬영 분야가 즉시 구분된다.
- 모바일 크롭에서도 핵심이 남는다.
- 텍스트가 이미지 안에 없다.
- 로고나 워터마크가 없다.
- 손가락/장비 형태가 심하게 깨지지 않는다.
- 같은 카테고리 이미지들이 너무 똑같지 않다.

불합격:

- 의미 없는 추상 배경
- 과한 보라/파랑 그라데이션
- 어두운 분위기만 있는 이미지
- stock photo 느낌이 강한 악수/회의 장면
- 촬영 장비가 보이지 않는 이미지
- 한글/영문 글자가 깨져 들어간 이미지

## 12. 구현 시 파일 구조 권장

프로젝트를 새로 만들 경우:

- `src/` 또는 `app/`: 화면 코드
- `components/`: 공통 UI
- `data/`: 샘플 데이터
- `public/images/presets/`: 이미지 프리셋
- `docs/`: 보조 문서

이미 프로젝트 구조가 있으면 기존 구조를 우선한다.

## 14. 검증 기준

구현 후 반드시 확인한다.

- PC와 모바일 모두 열리는지 확인한다.
- 주요 페이지를 Playwright 또는 브라우저로 캡처한다.
- 카드 텍스트가 넘치지 않는지 확인한다.
- 이미지가 깨지거나 비율이 틀어지지 않는지 확인한다.
- 비회원 연락처 잠금 상태가 보이는지 확인한다.
- 기업 인증 필요 흐름이 보이는지 확인한다.
- 프리미엄/추천/공지/마감 배지가 구분되는지 확인한다.
- 빠른 문의 버튼이 콘텐츠를 가리지 않는지 확인한다.
- 원본 편집몬 자산이 섞이지 않았는지 확인한다.
