# 실행/배포 가이드

로컬 실행, 검증, 외부 공유(터널), 배포 기준을 정합니다. (team2의 firebase-guide.md에 대응하는 문서 — 이 프로젝트는 Firebase를 쓰지 않습니다.)

## 목적

- 누구나 클론 후 바로 실행할 수 있게 합니다.
- 시연/공유 방식을 통일합니다.
- 배포 관련 결정을 한곳에 기록합니다.

## 기본값

### 로컬 실행

```bash
npm install
npm run dev     # http://localhost:3000
```

- 환경변수 없음 (.env 불필요). 백엔드/DB 연결 없음.
- 데모 조작은 좌하단 역할 스위처 사용.

### 검증 (PR 전 필수)

```bash
npm run build
npm run lint
```

- 둘 다 에러 0이어야 PR 가능. UI 변경 시 PC 1280/모바일 390 스크린샷 첨부.

### 외부 공유 (임시 시연)

- **Cloudflare 퀵 터널** 사용: `cloudflared tunnel --url http://localhost:3000`
  - 가입 불필요, 경고 페이지 없음, 원본 IP 비노출. URL은 실행마다 바뀜.
- ngrok은 사용하지 않습니다 — 무료 플랜 경고 페이지에 호스트 PC의 공인 IP가 노출됨 (2026-07-07 결정).
- 터널은 호스트 PC와 dev 서버가 켜져 있는 동안만 유효합니다.

### 정식 배포 (예정)

- 대상: **Vercel** (Next.js 공식 무료 호스팅, 고정 URL, PC 불필요).
- 배포 전 확인: 빌드 통과, 환경변수 없음 확인, `output/playwright/` 스크린샷 갱신.
- 배포 권한과 계정은 논의할 항목.

### 금지

- 프로덕션 빌드 설정을 검증 우회 목적으로 수정하지 않습니다 (`ignoreBuildErrors` 등).
- 실 서비스 배포 시 데모 역할 스위처(RoleSwitcher) 노출 여부를 팀 확인 후 결정합니다.

## 논의할 항목

- Vercel 계정 소유자와 배포 권한
- 배포 브랜치 전략 (main 자동 배포 여부)
- 커스텀 도메인 여부
