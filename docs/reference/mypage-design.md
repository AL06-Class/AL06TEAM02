# CLIPBee 마이페이지 설계

작성일: 2026-07-06
기준 문서: `PRD.MD` 13장, `architecture.md` 3.3(라우트)·6장(플로우), `wireframe.md`(공통 레이아웃), `ui-components.md`
범위: 개인회원/기업회원 마이페이지 전 화면. PC + 모바일.

## 1. 공통 구조

### 1.1 PC

```
┌사이드(220)─────────┐ ┌본문────────────────────────┐
│ ◯ 아바타            │ │ 페이지 제목                    │
│ 홍O민 · 개인회원      │ │ (화면별 콘텐츠)                │
│ 〈프로필 공개중〉      │ │                              │
│ ─────────────      │ │                              │
│ (역할별 메뉴 리스트)   │ │                              │
└────────────────────┘ └──────────────────────────────┘
```

- 사이드 상단: 아바타 + 마스킹 이름 + 회원 유형 + 핵심 상태 배지 1개
  - 개인: 프로필 상태 (공개중/검수중/미등록)
  - 기업: 인증 상태 (인증완료/검수중/미인증/반려)
- 현재 메뉴: primary 텍스트 + 좌측 3px 라인

### 1.2 모바일

- `/mypage` 진입 = 요약 화면 (프로필 카드 + 상태 요약 + 메뉴 리스트 세로 나열)
- 하위 화면은 개별 페이지로 이동, 상단에 ← 뒤로 + 화면 제목
- 표 형태 데이터는 카드 리스트로 전환 (ui-components 3.8)

### 1.3 게이트

- 비로그인 → `/login?redirect=/mypage`
- 역할에 없는 메뉴 접근 → `/mypage`로 리다이렉트 (예: 개인회원이 `/mypage/jobs`)

## 2. 개인회원 마이페이지

### 2.1 요약 대시보드 `/mypage`

```
┌내 프로필 카드────────────────────────────┐
│ ◯ 아바타  프로필 제목        〈활동가능〉〈추천〉│
│ 조회수 128 · 최종수정 2026-07-01           │
│ [프로필 수정] [미리보기]                    │
│ (미등록 시: EmptyState + [프로필 등록하기])  │
└─────────────────────────────────────────┘
┌지원 3건┐ ┌스크랩 5건┐ ┌받은 제안 1건┐ ┌내 상품 2건┐   ← 통계 카드 4개, 클릭 시 해당 메뉴
최근 활동: 지원/제안/결제 최신 5건 타임라인
```

### 2.2 내 프로필 관리 `/mypage/profile`

- 등록된 프로필 1건의 전체 필드 요약 표시 + [수정](→ `/profiles/new`와 동일 폼, 값 채움)
- 상태 전환: 공개 ↔ 비공개 토글, 현재상태 셀렉트(활동가능/일정협의/프로젝트중)
- 검수중/반려 상태면 상단 배너 (반려 사유 표시 + [수정 후 재요청])
- [삭제] — 확인 모달 (danger)

### 2.3 포트폴리오 관리 `/mypage/portfolio`

- 이미지 그리드 (드래그 정렬 2차, 1차는 순서 버튼 ↑↓) + [추가 업로드]
- 링크 목록: YouTube/Shorts/외부 행 + 미리보기 썸네일 + [추가]
- 각 항목 [삭제], 대표 지정 라디오

### 2.4 지원한 공고 `/mypage/applications`

| 컬럼 | 내용 |
| --- | --- |
| 상태 | 〈지원완료〉 / 〈열람됨〉(기업이 확인) / 〈마감〉 |
| 공고 | 회사명 + 제목 (링크) |
| 지원일 / 액션 | 날짜 / [지원 내용 보기](모달) |

빈 상태: "아직 지원한 공고가 없습니다" + [공고 탐색하기]

### 2.5 스크랩 공고 `/mypage/scraps`

JobRow 재사용 + 각 행 [스크랩 해제]. 마감 공고는 dim 처리.

### 2.6 내 스토어 상품 `/mypage/products`

ProductCard 축소형 리스트 + 상태 배지(검수중/판매중/반려/비공개) + [수정] [비공개 전환] [삭제]. 상단 [+ 상품 등록](→ `/store/new`)

### 2.7 추천 프로필 현황 `/mypage/promotion`

```
현재 상태: 〈노출중〉 2026-07-01 ~ 2026-07-31 (D-25)
노출 위치: 메인 추천 영역 · 프로필 목록 상단
누적 노출 기간: 2개월
[기간 연장 구매] → /services/order/promotion
(미구매 시: 상품 소개 카드 + 가격 + [신청하기])
```

### 2.8 받은 제안 (알림함 연동)

`/alerts` 지원/제안 탭에서 확인. 제안 행 클릭 → 제안 상세 모달 (기업명/관련 공고/메시지/[공고 보기]).

### 2.9 결제 내역 `/mypage/payments`

표: 날짜 / 상품명 / 금액 / 상태(결제완료·사용중·만료·환불) / [영수증](데모: 모달)

### 2.10 회원정보 수정 `/mypage/account`

비밀번호 확인 게이트(데모: 즉시 통과) → 이메일/휴대폰/닉네임 수정, 비밀번호 변경
하단 접힌 영역: [회원 탈퇴] — 사유 셀렉트 + 확인 문구 입력 모달 (danger)

## 3. 기업회원 마이페이지

### 3.1 요약 대시보드 `/mypage`

```
┌기업 카드─────────────────────────────────┐
│ 🏢 (주)새벽스튜디오        〈인증완료〉        │
│ (미인증/반려 시: warning/danger 배너 +      │
│  [인증하러 가기] — 이 배너는 모든 하위 화면 상단 공통) │
└──────────────────────────────────────────┘
┌진행 공고 2┐ ┌신규 지원자 5┐ ┌열람권 D-12┐ ┌점프 크레딧 23건┐  ← 클릭 시 해당 메뉴
최근 활동: 지원자/심사 결과/결제 타임라인
```

### 3.2 기업정보 관리 `/mypage/company`

회사명/대표자/업종/소재지/담당자/소개 수정 폼. 사업자등록번호는 읽기 전용(변경 = 재인증 안내). `/company/[id]` 공개 화면 [미리보기].

### 3.3 기업 인증 `/mypage/verification`

```
현재 상태: 〈검수중〉 (상태별 스텝퍼: 서류 제출 → 검수중 → 인증완료)
제출 서류: 사업자등록증명원.pdf (2026-07-03 제출)
안내: 최근 3개월 이내 발급 / 회사명·대표자·사업자번호 일치 필요
     사업자가 없는 개인 의뢰자는 본인 명의 통장 사본 대체 가능
반려 시: danger 배너 + 반려 사유 + [서류 다시 제출]
업로드: FileUpload (pdf/jpg/png, 10MB)  [제출하기]
```

### 3.4 공고 관리 `/mypage/jobs`

탭: 전체 | 게시중 | 심사중 | 반려 | 마감

| 컬럼 | 내용 |
| --- | --- |
| 상태 | 배지 (게시중/심사중/반려/마감) |
| 제목 | 링크 → 상세 |
| 지원자 | n명 (링크 → `/mypage/applicants?jobId=`) |
| 마감일 / 노출 | 날짜 / 〈프리미엄〉〈점프 ON〉 |
| 액션 | [수정] [점프](수동 점프 1회) [마감] [삭제] |

상단: [+ 공고 등록]. 반려 행: 사유 툴팁 + [수정 후 재심사].

### 3.5 지원자 관리 `/mypage/applicants`

좌측(PC): 내 공고 리스트(지원자 수 표시) → 선택 시 우측 지원자 목록

```
지원자 행: ◯ 김O민 · 프로필 제목 · 지원일 · 〈미확인/확인함〉
클릭 → 지원 상세 패널: 지원 내용 전문 + 프로필 카드 + 포트폴리오 링크
       [프로필 보기] [제안 보내기](열람권 게이트)
```

확인 시 지원자 측에 "열람됨" 상태 반영. 모바일: 공고 선택 → 지원자 목록 2단계 이동.

### 3.6 스크랩 프로필 `/mypage/scraps`

ProfileCard 그리드 + [스크랩 해제]. 카드에 연락처 열람 여부 표시(열람권 보유+열람 이력 시 "열람함").

### 3.7 연락처 열람권 `/mypage/contact-pass`

```
현재 이용권: 〈사용중〉 1주권 · 2026-07-03 ~ 07-10 (D-4)
만료 3일 전부터 warning 배너 + [연장 구매]
열람 로그 표: 날짜 / 촬영자(마스킹) / 프로필 링크   ← PRD 8.4 로그 요구
미보유 시: 상품 3종 가격 카드 + [구매하기]
```

### 3.8 자동점프 `/mypage/jump`

```
보유 크레딧: 23건   [크레딧 구매 →]
⚠ 자동점프는 구매 후 여기서 활성화해야 작동합니다 (PRD 12.1)
공고별 설정 표:
| 공고 제목 | 자동점프 [토글] | 다음 점프 예정 | 누적 사용 | [수동점프] |
자동점프 ON: 24시간마다 1건 차감 · 크레딧 0 = 자동 OFF + 알림
사용 내역: 날짜 / 공고 / 자동·수동 / 차감 후 잔여
```

### 3.9 프리미엄 배너 `/mypage/banners`

```
현재 배너: 〈집행중〉 2026-07-01 ~ 07-31 · 메인 상단 · 클릭 124회(데모 수치)
소재: ■ 미리보기 (940×230) + 〈검수중/승인/반려〉  [소재 교체]
클릭 URL: [__________] [저장]
미구매 시: 상품 안내 + [신청하기 → /services/order/banner]
```

### 3.10 결제 내역 / 회원정보 `/mypage/payments` `/mypage/account`

개인회원 2.9 / 2.10과 동일 구조. 결제 내역에 세금계산서 컬럼 자리만 확보(정책 미확정 — PRD 27장).

## 4. 상태·엣지 케이스 총괄

| 상황 | 처리 |
| --- | --- |
| 개인: 프로필 미등록 | 대시보드·지원 화면에서 등록 유도 EmptyState |
| 기업: 미인증 | 전 하위 화면 상단 warning 배너 + 공고 관련 메뉴에 자물쇠 아이콘 |
| 기업: 반려 | danger 배너 + 사유 + 재제출 버튼 |
| 열람권/추천/배너 만료 임박 (3일) | 해당 화면 배너 + 알림함 알림 |
| 크레딧 0 | 점프 토글 비활성 + [크레딧 구매] 강조 |
| 데이터 없음 | 화면별 EmptyState 문구 (ui-components 3.9) |

## 5. 모바일 요약 화면 메뉴 순서

개인: 내 프로필 → 지원한 공고 → 스크랩 → 받은 제안 → 포트폴리오 → 내 상품 → 추천 프로필 → 결제 내역 → 회원정보
기업: 인증 상태 → 공고 관리 → 지원자 → 열람권 → 자동점프 → 배너 → 스크랩 프로필 → 결제 내역 → 기업정보 → 회원정보
