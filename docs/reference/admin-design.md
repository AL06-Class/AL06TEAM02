# CLIPBee 관리자 화면 설계

작성일: 2026-07-06
기준 문서: `PRD.MD` 14장(기능)·17장(상태값), `architecture.md` 3.4(라우트), `design.md` 11장, `ui-components.md`
원칙: 운영 효율 우선. 테이블 중심, 화면당 [검색 + 필터 + 목록 + 상세 패널 + 액션] 구조 통일.

## 1. 공통 레이아웃

```
┌사이드바(240, ink 배경)─┐ ┌────────────────────────────────┐
│ CLIPBee 관리자           │ │ 상단 바: 통합검색 | 심사대기 🔔 | 관리자명  │
│ 대시보드               │ ├────────────────────────────────┤
│ ── 심사 ──            │ │ 페이지 제목 + 요약 통계 칩            │
│ 기업 인증 (2)          │ │ ┌필터 바: 상태 셀렉트+기간+검색────┐   │
│ 공고 심사 (3)          │ │ ├데이터 테이블───────────────────┤   │
│ 프로필 검수 (1)        │ │ │ ☐ | 상태 | 제목/이름 | … | 액션    │   │
│ 포트폴리오 검수         │ │ └───────────────────────────────┘   │
│ 스토어 검수 (1)        │ │ 페이지네이션                          │
│ ── 운영 ──            │ │                    ┌상세 패널(우측 슬라이드)┐
│ 회원 관리              │ │                    │ 선택 항목 상세 + 액션   │
│ 게시판 관리            │ │                    └──────────────────┘
│ 신고 관리 (2)          │ └────────────────────────────────────┘
│ 결제 관리 / 배너·노출    │
│ 카테고리 / 공지사항      │
└──────────────────────┘
```

- 사이드바 메뉴 옆 숫자 = 처리 대기 건수 배지 (warning)
- **상세 패널**: 행 클릭 시 우측 480px 슬라이드 (모달 아님 — 목록 유지). 하단에 액션 버튼 고정
- 승인 = primary / 반려 = danger + **사유 입력 필수** / 보완요청 = secondary + 요청 내용 입력
- 모든 승인/반려/수정은 AdminAuditLog에 기록된다는 전제 (데모: localStorage)
- 접근: `admin` 역할 외에는 404 (architecture.md 5장)

## 2. 대시보드 `/admin`

```
1행 — 처리 대기 카드 5개 (클릭 시 해당 화면):
  [기업 인증 2] [공고 심사 3] [프로필 검수 1] [스토어 검수 1] [신고 2]
2행 — 오늘 요약: 신규 가입 n · 신규 공고 n · 신규 지원 n · 매출 n원(데모)
3행 — 최근 신고 5건 리스트  |  최근 결제 5건 리스트 (2단)
```

## 3. 심사 화면 5종 (공통 패턴 + 화면별 차이)

공통 테이블 컬럼: ☐ | 상태 | 대상 | 신청자/작성자 | 접수일 | 액션
공통 액션: [승인] [반려(사유 필수)] / 일괄 선택 승인 지원

### 3.1 기업 인증 심사 `/admin/verifications`

- 필터: 미인증/검수중/인증완료/반려/만료
- 상세 패널: 회사명·대표자·사업자번호 (가입 정보) ↔ 제출 서류 뷰어(■ 미리보기) **나란히 대조**
- 체크리스트 UI: ☐ 3개월 이내 발급 ☐ 회사명 일치 ☐ 대표자 일치 ☐ 사업자번호 일치
- 액션: [승인] [반려] [보완요청] — 반려 사유 프리셋(서류 기한 초과/정보 불일치/판독 불가/기타 — 코드화는 PRD 27장 미확정, 프리셋으로 우선 처리)

### 3.2 공고 심사 `/admin/jobs`

- 필터: 심사중/게시중/반려/마감/비공개 + 프리미엄 여부
- 상세 패널: 공고 전문 미리보기 (실제 상세 페이지와 동일 렌더) + 기업 인증 상태 표시
- 점검 포인트 안내: 불건전/저단가/연락처 오기재/광고성 (신고 사유와 동일 축)
- 액션: [승인→게시중] [반려] [비공개 전환] (게시중 공고도 사후 조치 가능)

### 3.3 프로필 검수 `/admin/profiles`

- 필터: 검수중/공개/비공개/반려
- 상세 패널: 프로필 전문 + 연락처 필드(관리자는 열람 가능) + 포트폴리오 썸네일
- 액션: [승인→공개] [반려] [비공개 전환]

### 3.4 포트폴리오 검수 `/admin/portfolios`

- 대상: 프로필/게시글에 첨부된 YouTube/Shorts/외부 링크
- 테이블: 링크 | 소유자 | 유형 | 등록일 | 상태(정상/재생불가/부적절)
- 상세 패널: 임베드 플레이어 + [정상] [차단(사유)] — 차단 시 프로필에서 해당 링크만 숨김
- 저작권/초상권 확인 안내 문구 노출 (PRD 24장)

### 3.5 스토어 검수 `/admin/store`

- 필터: 검수중/판매중/반려/비공개
- 상세 패널: 상품 전문 + 환불 규정 존재 여부 자동 체크 표시 + 판매자 프로필 링크
- 액션: [승인→판매중] [반려] [비공개]
- 하위 탭: 주문/환불 관리 (2차 — 자리만 확보, "2차 오픈 예정" EmptyState)

## 4. 운영 화면

### 4.1 회원 관리 `/admin/members`

- 탭: 개인회원 | 기업회원
- 컬럼: 상태(정상/휴면/정지/탈퇴) | 아이디 | 이름/회사명 | 가입일 | 최근 접속 | 액션
- 상세 패널: 회원 정보 + 활동 요약(공고/프로필/지원/결제 수) + 신고당한 이력
- 액션: [정지(기간+사유)] [정지 해제] [강제 탈퇴(확인 모달)] — 개인정보 필드는 마스킹 기본, [전체 보기] 클릭 시 감사 로그 기록 안내

### 4.2 게시판 관리 `/admin/boards`

- 상단: 게시판 선택 탭(8종) + 검색
- 게시글 테이블: 상태 | 제목 | 작성자 | 댓글수 | 조회 | 등록일 | 액션([숨김] [삭제] [공지 지정/해제] [이동])
- 댓글 하위 탭: 신고된 댓글 우선 정렬, [숨김] [삭제]
- 공지 지정 시 해당 게시판 상단 고정 + 〈공지〉 배지

### 4.3 신고 관리 `/admin/reports`

- 필터: 접수/검토중/조치완료/반려 + 대상 유형(공고/프로필/게시글/댓글/상품)
- 컬럼: 상태 | 유형 | 대상(링크) | 사유 | 신고자 | 접수일
- 상세 패널: 신고 내용 전문 + 대상 미리보기 + 동일 대상 누적 신고 수
- 액션: [검토 시작] → [조치완료(조치 내용 선택: 숨김/삭제/경고/회원 정지)] 또는 [반려(사유)]
- 처리 결과는 신고자 알림으로 발송 (F7 플로우)

### 4.4 결제 관리 `/admin/payments`

- 탭 1 결제 내역: 날짜 | 회원 | 상품 | 금액 | 상태(결제완료/취소/환불요청/환불완료) | 액션([환불 승인] [환불 거절(사유)])
- 탭 2 **상품 가격 관리** (PRD 12.1 "가격은 관리자에서 수정 가능"):
  - 상품 4종(배너/점프/열람권/추천) × 옵션별 가격 인라인 편집 + [저장]
  - 변경 이력 테이블 (이전 가격 → 새 가격, 변경자, 일시)

### 4.5 배너/노출 관리 `/admin/banners`

- 탭 1 프리미엄 배너: 슬롯 순서 리스트(↑↓) | 광고주 | 소재 ■ | 기간 | 상태(검수중/집행중/만료) | [승인] [반려] [기간 수정]
- 탭 2 추천 프로필 슬롯: 슬롯 수 설정(기본 6) + 노출 순서 ↑↓ + 기간 관리 (PRD 12.2)
- 탭 3 메인 배너/팝업: 자체 운영 배너 등록(이미지+링크+기간) + 팝업 관리(노출 기간/오늘 하루 보지 않기)

### 4.6 카테고리 관리 `/admin/categories`

- 탭: 촬영 분야(14) | 장비/스킬(13) | 지역 | 역세권 | 스토어 카테고리(9)
- 각 탭: 항목 리스트 + 사용 건수 + [추가] [이름 수정] [숨김] (삭제는 사용 건수 0일 때만)
- 순서 변경 ↑↓ — 목록 필터 노출 순서에 반영

### 4.7 공지사항 작성 `/admin/notices`

- 목록: 고정 여부 | 제목 | 등록일 | 조회 | 액션
- 작성 폼: 제목 / 본문 에디터 / ☐ 상단 고정 / 노출 채널(사이트·알림) / [게시]

### 4.8 설정 (2차 자리 확보) `/admin/settings`

자동등록방지(스팸 차단), RSS/외부 피드, 알림 템플릿 — 1차는 메뉴만 두고 "2차 오픈 예정" EmptyState.

## 5. 상태값 · 배지 (PRD 17장 ↔ 관리자 화면 매핑)

| 도메인 | 상태 흐름 | 액션 주체 |
| --- | --- | --- |
| 기업 인증 | 미인증 → 검수중 → 인증완료 / 반려 → (재제출) → 검수중 / 만료 | 3.1 |
| 공고 | 심사중 → 게시중 / 반려 · 게시중 → 마감/비공개 | 3.2 |
| 프로필 | 검수중 → 공개 / 반려 · 공개 ↔ 비공개 | 3.3 |
| 스토어 | 검수중 → 판매중 / 반려 · 판매중 ↔ 비공개 | 3.5 |
| 신고 | 접수 → 검토중 → 조치완료 / 반려 | 4.3 |
| 결제 | 결제완료 → 사용중 → 만료 · 환불요청 → 환불완료/거절 | 4.4 |

## 6. 데모 구현 범위

1차 (관리자 심사 시연 필수 — AGENTS.md 6장):
- 대시보드, 기업 인증 심사, 공고 심사, 프로필 검수, 신고 관리
- 승인/반려는 localStorage 상태 변경으로 사용자 화면에 즉시 반영 (예: 반려 → 기업 마이페이지에 반려 배너)

2차: 포트폴리오 검수, 회원 관리 상세, 결제/가격 관리, 배너 슬롯, 카테고리 관리, 설정
