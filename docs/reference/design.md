# 촬영몬 디자인 문서

작성일: 2026-07-06  
기준 문서: `PRD.MD`  
목표: 촬영몬을 만들 때 필요한 PC/모바일 화면 구조, 디자인 시스템, 컴포넌트, 상태, 페이지별 UI 규칙을 정리한다.

## 1. 디자인 목표

촬영몬은 촬영 의뢰자와 촬영자를 빠르게 연결하는 실무형 플랫폼이다. 디자인은 화려한 랜딩페이지보다 구인구직, 프로필 탐색, 연락처 열람, 지원, 결제 흐름이 명확해야 한다.

핵심 방향:

- 첫 화면에서 바로 공고와 촬영자 프로필이 보여야 한다.
- 촬영 분야 플랫폼답게 이미지와 포트폴리오를 크게 보여준다.
- 검색, 필터, 정렬은 한눈에 읽혀야 한다.
- PC는 정보 밀도, 모바일은 카드 가독성을 우선한다.
- 유료 노출 상품은 광고처럼 보이되, 목록 탐색을 방해하지 않는다.
- 편집몬의 기능 구조는 반영하지만, 시각 자산과 브랜드 느낌은 촬영몬 고유로 만든다.

## 2. 브랜드 방향

### 2.1 브랜드 키워드

- 현장감
- 신뢰
- 빠른 매칭
- 포트폴리오 중심
- 전문 촬영 인력
- 제작 실무

### 2.2 톤

문구는 짧고 실무적으로 쓴다.

좋은 예:

- 촬영자 모집
- 촬영자 프로필
- 포트폴리오 보기
- 온라인 지원
- 연락처 열람
- 기업 인증 필요
- 프리미엄 노출 중

피해야 할 예:

- 과장된 마케팅 문구
- 너무 긴 기능 설명
- 사용법을 화면 안에 길게 풀어쓴 안내
- 원본 사이트 문구 그대로 복사

### 2.3 로고 방향

권장:

- 카메라 렌즈, 뷰파인더, 프레임, 녹화 점, 조리개 중 하나를 단순화한다.
- 텍스트 로고는 `촬영몬`이 첫 화면에서 분명히 보이게 한다.
- 모바일 헤더에서는 아이콘과 짧은 워드마크가 함께 보이는 크기로 설계한다.

금지:

- 편집몬 로고 색상/형태 직접 복제
- 기존 이미지 자산 재사용
- 과도한 3D 아이콘 남발

## 3. 색상 시스템

### 3.1 기본 팔레트

| 토큰 | 용도 | 색상 |
| --- | --- | --- |
| Primary | 주요 버튼, 활성 탭, 강조선 | `#1276B1` |
| Primary Dark | hover, 눌림 상태 | `#0D5E8E` |
| Accent | 추천/프리미엄 보조 강조 배경 | `#DBF15F` |
| Success | 채용시, 인증완료, 공개 | `#10B981` |
| Warning | 심사중, 만료 예정, 공지 | `#F59E0B` |
| Danger | 신고, 반려, 마감 | `#EF4444` |
| Ink | 본문 주요 텍스트 | `#111827` |
| Muted | 보조 텍스트 | `#6B7280` |
| Line | 경계선 | `#E5E7EB` |
| Surface | 카드 배경 | `#FFFFFF` |
| Page | 페이지 배경 | `#F8FAFC` |

### 3.2 색상 사용 규칙

- Primary는 촬영몬의 주 행동에만 쓴다.
- 목록 카드 전체를 Primary 계열로 칠하지 않는다.
- 배지는 기능별 색상만 사용한다.
- 배경은 흰색과 아주 연한 회색 위주로 유지한다.
- 프리미엄/추천 상품은 Accent 또는 Warning을 작은 배지로 표현한다.
- Accent Lime 위 텍스트는 `Ink #111827`을 사용한다.

## 6. 공통 컴포넌트

### 6.1 헤더 PC

구성:

- 좌측 로고
- 우측 아이콘 바로가기: 광고배너 안내, 서비스안내, 마이페이지, 고객센터
- 하단 주 메뉴: 촬영자 모집, 촬영자 프로필, 커뮤니티, 공지사항, 스토어, 스토어 등록
- 우측 통합 검색
- 메뉴 하단 Primary 색상 라인

상태:

- 현재 메뉴는 텍스트 색상과 하단 라인으로 강조한다.
- 검색창 focus 시 Primary 테두리를 적용한다.

### 6.2 헤더 모바일

구성:

- 좌측 햄버거
- 중앙 로고
- 우측 마이페이지 아이콘
- 검색 입력
- 가로 탭 메뉴

상태:

- 현재 탭은 하단 Primary 라인
- 햄버거 클릭 시 전체 메뉴 드로어
- 드로어는 좌측에서 열리고 배경 dim 처리

### 6.3 버튼

버튼 종류:

- Primary: 검색, 로그인, 지원, 결제
- Secondary: 목록, 취소, 뒤로
- Outline: 필터, 페이지 번호
- Danger: 신고, 삭제
- Kakao: 카카오 로그인

규칙:

- 버튼 높이 PC 40px, 모바일 44px
- radius 8px 이하
- 같은 줄에 주요 행동은 하나만 둔다.
- 아이콘이 의미를 더하면 아이콘과 텍스트를 함께 쓴다.

### 6.4 카드

공통:

- 배경 흰색
- border `#E5E7EB`
- radius 8px
- 그림자 약하게
- hover 시 테두리 또는 그림자만 강화

금지:

- 카드 안에 또 큰 카드를 넣지 않는다.
- 카드 전체에 강한 색 배경을 쓰지 않는다.

### 6.5 배지

배지 종류:

- 프리미엄
- 추천
- 채용시
- 상시채용
- 마감
- 경력무관
- 인증완료
- 심사중
- 반려
- 신규
- 공지

규칙:

- 배지는 작은 정보 보조 수단이다.
- 배지 1개당 한 줄을 넘기지 않는다.
- 공고 카드에는 최대 4개까지만 노출한다.

### 6.6 검색/필터

PC:

- 상단 필터 박스는 카드형으로 묶는다.
- 체크박스, 셀렉트, 키워드 입력, 검색 버튼을 한 영역에 배치한다.
- 좌측 카테고리는 항상 유지한다.

모바일:

- 상세검색은 별도 페이지로 이동한다.
- 필터 결과 적용 후 목록으로 돌아온다.
- 선택된 필터는 목록 상단 칩으로 표시한다.

### 6.7 페이지네이션

PC:

- 1~10 번호
- 이전/다음
- 현재 페이지는 Ink 배경, 흰색 텍스트

모바일:

- 1~5 번호
- 다음 버튼
- 터치 가능한 크기 유지

## 8. 모바일 페이지 설계

## 10. 상태 화면

### 10.1 빈 상태

예시:

- 등록된 촬영자 모집이 없습니다.
- 조건에 맞는 촬영자 프로필이 없습니다.
- 아직 등록된 상품이 없습니다.

빈 상태에는 한 가지 행동만 제공한다.

- 공고 등록하기
- 프로필 등록하기
- 필터 초기화

### 10.2 로딩

- 목록은 스켈레톤 카드
- 상세는 제목/이미지/본문 스켈레톤
- 결제/지원 제출은 버튼 내부 로딩

### 10.3 오류

- 네트워크 오류
- 로그인 필요
- 권한 없음
- 기업 인증 필요
- 결제 실패
- 파일 업로드 실패

오류 문구는 원인과 다음 행동을 함께 보여준다.

## 12. 접근성

필수:

- 버튼은 키보드 포커스가 보여야 한다.
- 이미지에는 대체 텍스트를 둔다.
- 색상만으로 상태를 구분하지 않는다.
- 텍스트 대비는 충분히 확보한다.
- 모바일 터치 영역은 최소 44px 높이로 둔다.
- 폼 오류는 스크린리더가 읽을 수 있어야 한다.

## 18. 체크리스트

완성 전 확인:

- 모바일에서 제목이 카드 밖으로 넘치지 않는다.
- 공고 카드와 프로필 카드가 구분된다.
- 프리미엄/추천/공지 배지가 혼동되지 않는다.
- 비회원 연락처 잠금이 명확하다.
- 기업 인증 필요 상태가 명확하다.
- 검색과 필터가 PC/모바일 모두 접근 가능하다.
- 빠른 문의 버튼이 본문을 가리지 않는다.
- 푸터 사업자/약관 링크가 모든 페이지에 있다.
- 원본 사이트 로고/이미지/문구를 그대로 쓰지 않았다.
